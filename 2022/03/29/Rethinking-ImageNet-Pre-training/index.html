Rethinking ImageNet Pre-training

Abstract

We report competitive results on object detection and instance segmentation on the COCO dataset using standard models trained from random initaliation. The result are no worse than their ImageNet pre-training counterparts even when using the hyper-parameters of the baseline system(Mask R-CNN) that were optimized for fune-tuin pretrained models, with the sold exception of increasing the number of training iterations so the randomly initalized models may converge.
Training from random initalization is superisingly robust;
Our results hold even when:

  1. using only 10% of the training data
  2. for deeper and wider models
  3. for multiple tasks and metrics
    Experiments shows that ImageNet pre-training speeds up convergence early in training, but does not necessarily provide regularization or improve final target task accuracy.

These observations challenge the conventional wisdom of ImageNet pre-training for dependent tasks and we expect these discoveries will encourage people to rethink the current de fcto paradigm of “pre-training and fine-tuning” in computer vision.

1. Introduction

In recent years, a well-established paradigm has been to pre-train models using large-scale data and then fine-tune the models on target tasks that often have less training data.

Pre-training has enabled state-of-the-art results on many tasks, including object detection, image segmentation, and action recognitaion.

This papaer questions the paradigm of pre-training even further by exploring the opposite regime: we report that competitive object detection and instance segmentation accuracy is achievable when training on COCO from random initalization (from scratch), without any pre-training. More surprisingly, we can achieve these results by using basline systems and their hyper-parameters that there is no fundamental obstacle preventing us from training from scratch, if:

  1. we use normalization technique appropriately for optimization
  2. we train the models sufficiently long to compensate for the lack of pre-trianing.

Based on these experiments and others, we observe the following:

  1. ImageNet pre-training speeds up convergence, especially early on in training, but trainin from random initalizaiton can catch up after training for a duration.
  2. ImageNet pre-training does not automatically give better regularization.
  3. ImageNet pre-training shows on benefit when the target tasks/metrics are more sensitive to spatially well localized predictions. Intuitively, the task gap between the classification-based, ImageNet like pre-training and localization-sensitive target tasks may limit the benefits of pre-training.

3. Methodology

We decribe the only two modifications that we find to be necessary, related to model normalization and training length, discussed next.

3.1 Normalization

Successful forms of normalization include normalized parameter initalization and activation normalization layers.

Batch Normalization, the popular normalization method used from train modern networks, partically makes traning detectors from scratch difficult.

Object detectors are typically trained with high resolution inputs, unlike image classifiers. This reduces batch sizes as constrained by memory, and small batch sizes severely degrade the accuracy of BN. This issue can be circumvented if pre-training is used, bacause fine-tuning can adopt the pre-training batch statistics as fixed parameters; however, freezing BN is invalid when training from scratch.

We also report that using appropriately normalization initalization, we can train object detectors with VGG nets from random initalizaiton with out BN or GN.

6. Disucssions

We summarize the main observations from our experiments as follows:

  • Training from scratch on target tasks is possible with architecture change.
  • Training from scratch on scratch requires more iterations to sufficiently converge.
  • Training from scratch can be no worse than its ImageNet per-training counterparts under many circumstances down to as few as 10k COCO images.
  • ImageNet pre-training speeds up convergence on the target task.
  • ImageNet pre-training does not necessarily help reduce overfitting unless we enter a very small data regime.
  • ImageNet pre-training helps less if the target task is more sensitive to localization than classification.

Based on these observations, we provide our answers to a few important questions that may encourage people to rethink ImageNet pre-training:

Is ImageNet pre-training necessary? No -if we have enought target data.

Is ImageNet helpful? Yes. ImageNet pre-training has been a critical auxiliary task for computer vision community to progress.

Do we need big data? Yes. But a generic large scale, classification-level pre-training set is not ideal if we take into account the extra effort of collecting and cleaning data – the cost of collecting ImageNet has been largely ignored, but the ‘pre-training’ step in the ‘pre-training + fine-tuning’ paradigm is in fact not free when we scale out this paradigm.

Shall we pursuit universal representations? Yes. We believe learning universal representations is a laudable goal.
